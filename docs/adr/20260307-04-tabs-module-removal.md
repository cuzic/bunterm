# ADR 073: タブモジュール削除（ADR 034 を置換）

## ステータス

採用（ADR 034: Session Tabs を**置換**）

## コンテキスト

ADR 034 で導入したセッションタブ機能は、ブラウザ UI 上部にタブバーを表示し、複数セッション間の切り替えを提供していた。しかし、実運用を通じていくつかの問題が明らかになった。

### 問題点

1. **モバイル UI の制約**: タブバーが画面上部のスペースを占有し、スマートフォンの狭い画面でターミナル表示領域が圧迫される
2. **タッチ操作の困難さ**: 小さなタブをタップする操作がモバイルで困難
3. **実装の複雑さ**: タブの状態管理、ドラッグ&ドロップによる並び替え、タブのクローズ処理など、UI コードが肥大化
4. **設定の肥大化**: `TabsConfig` 型が config.yaml に追加され、設定スキーマが複雑化

### SessionSwitcher モーダルの登場

タブの代替として、モーダルダイアログによるセッション切り替え機能（SessionSwitcher）が実装された。モーダルは必要なときだけ表示され、画面スペースを常時占有しない。

## 決定

### タブモジュールの完全削除

以下を削除する:

1. **tabs モジュール**: ブラウザ側のタブ UI コンポーネント一式
2. **TabsConfig**: 設定ファイルからタブ関連の型と設定項目を削除
3. **テストフィクスチャ**: タブ設定を含むテストフィクスチャを更新

### SessionSwitcher モーダルで代替

- キーボードショートカットまたはツールバーボタンでモーダルを開く
- セッション一覧をリスト表示し、選択で切り替え
- モバイルでもフルスクリーンモーダルで快適に操作可能

## 代替案

### タブ UI の改善（レスポンシブ対応）

**採用しなかった理由**:
- モバイルでタブバーをハンバーガーメニューに折りたたむと、結局モーダルと同じ UX になる
- 常時表示のタブバーはデスクトップでもスペース効率が悪い

### タブとモーダルの併用

**採用しなかった理由**:
- 2 つの UI パスを維持するコスト
- ユーザーが混乱する（どちらを使えばいいか不明確）

## 影響

### Positive

- **UI のシンプル化**: 常時表示の UI 要素が減り、ターミナル表示領域が最大化
- **モバイル体験の向上**: モーダルはタッチ操作に適している
- **コードの削減**: タブ関連のコンポーネント、状態管理、設定型が不要に
- **設定の簡素化**: TabsConfig が config.yaml から消え、設定スキーマが単純化

### Negative

- **常時可視性の喪失**: タブバーは現在のセッションを常時表示していたが、モーダルでは能動的に開く必要がある
