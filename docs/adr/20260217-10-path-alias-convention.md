# ADR 071: パスエイリアス @/ 規約

## ステータス

採用

## コンテキスト

bunterm のコードベースが成長するにつれ、相対パスによるインポートが深くネストされるようになった。

```typescript
// 典型的な問題例
import { loadConfig } from "../../../core/config/config.js";
import { SessionManager } from "../../../../core/server/session-manager.js";
```

### 問題点

1. **可読性の低下**: `../../../..` の連鎖でインポート先が直感的に把握できない
2. **リファクタリングの困難さ**: ファイルを移動すると相対パスがすべて壊れる
3. **一貫性の欠如**: 同じモジュールへの参照がファイルの場所によって異なるパスになる

## 決定

### tsconfig.json に `@/` エイリアスを追加

`@/` を `src/` ディレクトリのエイリアスとして定義し、すべてのインポートを変換する。

```json
// tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

#### 変換例

```typescript
// Before
import { loadConfig } from "../../../core/config/config.js";

// After
import { loadConfig } from "@/core/config/config.js";
```

### 全インポートの一括変換

コミット 6387e5f で、既存の親ディレクトリ相対インポート（`../` を含むもの）をすべて `@/` エイリアスに変換した。

### 規約

- **プロジェクト内インポート**: 常に `@/` エイリアスを使用
- **相対インポート**: 同一ディレクトリ内（`./`）のみ許可
- **Node モジュール**: `node:` プロトコルを使用（`node:fs`, `node:path` 等）

## 代替案

### `~` エイリアス

**採用しなかった理由**:
- `@/` が TypeScript / Vite / Next.js エコシステムで事実上の標準
- チーム間での認知負荷が低い

### baseUrl のみ（エイリアスなし）

```json
{ "compilerOptions": { "baseUrl": "./src" } }
```

**採用しなかった理由**:
- `import { x } from "core/config/config.js"` がパッケージ名と紛らわしい
- `@/` プレフィックスで「プロジェクト内パス」であることが明示される

### 変換しない（相対パス維持）

**採用しなかった理由**:
- 上述の問題が解消されない
- コードベースの成長に伴い悪化する一方

## 影響

### Positive

- **可読性向上**: インポート先がパスから即座に判別可能
- **リファクタリング容易**: ファイル移動時に `@/` パスは変更不要（ディレクトリ構造が変わらない限り）
- **一貫性**: どのファイルからも同じパスで同じモジュールを参照

### Negative

- **Bun の互換性**: Bun がパスエイリアスをネイティブ解決するため問題ないが、他のツールでは追加設定が必要な場合がある
- **一括変換の影響範囲**: 多数のファイルが変更されるため、コンフリクトのリスク
